# Desarrollo de Software*

* Aplicar siempre un **ciclo de vida** al desarrollo software*


**Existe una serie de etapas a seguir:**

> Análisis

>> Diseño

>>>Codificación

>>>> Pruebas

>>>>> Mantenimiento

.........................


> 2.1 Fases principales: Análisis

- Determina las necesidades del cliente + **especifica requisitos** (Desarrollo)

**Especificación requisitos**

- Completa sin omisiones 
- Concisa
- Lenguaje formal
- Evita detalles diseño e implementación
- Entendible por cliente
- Requisitos funcionales y no funcionales
- Dividir + jerarquizar el modelo
- Fijar criterior de validación

.........................

>> Fases principales: Diseño

- Se descompone y organiza el sistema

- Se especifica la interrelación y funcionalidad de los  elementos.

**Las actidades habituales son las siguientes:**

- Diseño Arquitectónico

- Diseño detallado

- Diseño de datos

- Diseño de interfaz de usuario

...........

>>> Fases Principales: Codificación

- Escribir código fuente (componente)


 Uso de distintos lenguajes informáticos:

 -> **Lenguajes de programación:** C, C++, Java, Javascript, ...

 -> **Lenguajes de otro tipo:** HTML, XML, JSON...

 .........

 >>>> Fases principales: Pruebas


 - **Doble funcionalidad**, pruebas buscan confirmar que la codificación ha sido exitosa, sin errores de software

 >>>>> Fases principales: Mantenimiento

 - Hay que rehacer parte del trabajo realizado en fases previas.

 **Tipo de mantenimiento:**

 --> Correctivo: se corrigen defectos.

 --> Perfectivo: se mejora la funcionalidad.

 --> Evolutivo: se añade funcionalidades nuevas.

 -> Adaptativo: se adapta a nuevos entornos.

 ............

# Modelo en cascada o clásico

- Modelo de mayor antiguedad.

- Identifica **fases principales** del desarrollo software.

- Fases, en el orden indicado anteriormente.

- Modelo bastante rígido que se adapta mal al cambio contínuo

- Existen diferentes variantes con mayor o menor cantidad de actividades.


$ Modelo - Cascada $

> Comunicación

    - Inicio del proyecto

    - Recopilación de requisitos


>> Planeación

    - Estimación
    - Itinerario
    - Seguimiento


>>> Modelado

>>>> Construcción

>>>>> Despliegue

# Modelo en V

* Similar a modelo Cascada

* Visión jerarquizada con distintos niveles.

* Niveles superiores = mayor abstracción

* Niveles inferiores = mayor nivel (detalle)

--> ***Resultado:** *entrada de la siguiente fase*
